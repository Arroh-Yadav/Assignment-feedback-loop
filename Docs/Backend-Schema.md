# Backend Schema — Assignment Feedback Loop
## IPS Academy, Indore

---

## Overview

The database has **12 tables** built on PostgreSQL (hosted on Supabase).
All IDs are UUIDs. Enums are used for role and status fields.

---

## Table Relationship Summary

```
USERS
  ├── STUDENTS → SECTIONS → SUB_BRANCHES → BRANCHES
  └── FACULTY  → BRANCHES
                    └── FACULTY_SECTIONS → SECTIONS

ASSIGNMENTS (created by FACULTY for a SECTION)
  └── SUBMISSIONS (by STUDENTS)
        ├── AI_FEEDBACK (auto-generated by Gemini)
        └── EVALUATIONS (by FACULTY)
              └── ANNOTATIONS
```

---

## 1. `users` — Master auth table

Every person on the platform has one row here regardless of role.

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Primary key |
| `role` | ENUM (`student`, `faculty`, `admin`) | User type |
| `employee_or_computer_code` | VARCHAR | Login identity |
| `enrollment_or_email` | VARCHAR | Second login credential |
| `is_active` | BOOLEAN | Synced from institute DB |
| `created_at` | TIMESTAMP | First login time |
| `last_login` | TIMESTAMP | Session tracking |

| Field | Type | Notes |

| `faculty_id` | UUID | FK → faculty |
| `section_id` | UUID | FK → sections |
| `subject_name` | VARCHAR | e.g. Machine Learning |

## 8. `assignments` — Created by faculty

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Primary key |
| `faculty_id` | UUID | FK → faculty |
| `section_id` | UUID | FK → sections |
| `title` | VARCHAR | Assignment name |

| `subject_name` | VARCHAR | Subject it belongs to |
| `max_marks` | INT | Total marks |
| `deadline` | TIMESTAMP | Due date/time |
| `status` | ENUM (`draft`, `active`, `closed`) | Publish state |
| `reference_file_url` | VARCHAR | Optional Cloudinary URL |
| `created_at` | TIMESTAMP | |

---

## 9. `submissions` — Student uploads

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Primary key |
| `assignment_id` | UUID | FK → assignments |
| `student_id` | UUID | FK → students |
| `status` | ENUM (`submitted`, `ai_processing`, `ai_done`, `under_review`, `graded`) | Pipeline state |
| `file_urls` | TEXT[] | Array of Cloudinary URLs per page |
| `page_count` | INT | Number of pages uploaded |
| `student_note` | TEXT | Optional note to faculty |

---

## 10. `ai_feedback` — Gemini's analysis result

One row per submission, generated automatically after upload.

| Field | Type | Notes |
|---|---|---|
| `id` | UUID | Primary key |
| `submission_id` | UUID | FK → submissions (unique) |
| `extracted_text` | JSONB | What Gemini read from handwriting |
| `structural_analysis` | JSONB | Step-by-step method check |
| `feedback_items` | JSONB | Array of `{type, message}` objects |
| `suggested_marks` | INT | Gemini's mark suggestion |
| `model_used` | VARCHAR | e.g. `gemini-1.5-pro` |
| `generated_at` | TIMESTAMP | |

### `feedback_items` JSON structure
```json
[
  { "type": "correct", "message": "Chain rule applied correctly in step 2." },
  { "type": "error",   "message": "Learning rate term missing from gradient formula." },
  { "type": "warning", "message": "Diagram labels are unclear — ensure legibility." }
]
```

---

## Storage Strategy

| What | Where |
|---|---|
| Handwritten pages (images/PDFs) | Cloudinary (CDN) |
| File URLs | Stored in `submissions.file_urls` array |
| All structured data | PostgreSQL via Supabase |
| Auth verification | Read-only sync from IPS Academy cloud DB |
| Background job queue | Upstash Redis |

## Enums Reference

```sql
CREATE TYPE user_role AS ENUM ('student', 'faculty', 'admin');

CREATE TYPE submission_status AS ENUM (
  'submitted', 'ai_processing', 'ai_done', 'under_review', 'graded'
);

CREATE TYPE assignment_status AS ENUM ('draft', 'active', 'closed');

CREATE TYPE evaluation_status AS ENUM ('in_review', 'published');

CREATE TYPE annotation_type AS ENUM ('highlight', 'comment', 'drawing');
```
